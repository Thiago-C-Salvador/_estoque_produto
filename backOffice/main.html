Supracitado as informações no parágrafo anterior e me apresentado, irei resumir quais as funções do sistema, que são:

Colaborador (usuário)
  • Data Grid View (DGV) com a lista de colaboradores (inicialmente apenas os ativos);
  • Campo de busca por nome de colaborador;
  • Busca pelo último ID de colaborador;
  • Busca por colaboradores dentro de um intervalo de ID's;
  • Botão adicionar novo colaborador(usuário):
    • Novo colaborador a ser cadastrado não poderá ter o mesmo user name de outro colaborador;
      • User Name será um endereço de e-mail.
    • Todos os campos são obrigatórios no cadastro de colaborador, exceto carregamento da foto;
    • Deverá ser informado ao menos um contato telefônico para o colaborador.
  • Botão listar todos os colaboradores no DGV (ativos e inativos);
  • Botão exibir todos os colaboradores caso tenha resuzido a lista devido a uma pesquisa;
  • Botão atualizar lista de colaboradores - em caso de mais de um usário utilizar simultaneamente o sistema e precisar da lista atualizada;
  • DGV:
    • O DGV terá uma coluna com operações(funções):
      • Ativar e desativar usuário(colcaborador);
      • Vizualizar contato(s) do colaborador;
      • Editar colaborador;
      • Excluir colaborador.
Fornecedor
  • Data Grid View (DGV) com a lista de fornecedores (inicialmente apenas os ativos);
  • Campo de busca por nome de fornecedor;
  • Busca pelo último ID de fornecedor;
  • Busca por fornecedores dentro de um intervalo de ID's;
  • Botão adicionar novo fornecedor:
    • Novo fornecedor:
      • Deverá conter ao menos um contato telefônico da empresa e um contato telefônico do representante.
    • Todos os campos são obrigatórios no cadastro de fornecedor;
  • Botão listar todos os fornecedores no DGV (ativos e inativos);
  • Botão exibir todos os fornecedores caso tenha resuzido a lista devido a uma pesquisa;
  • Botão atualizar lista de fornecedores - em caso de mais de um usário utilizar simultaneamente o sistema e precisar da lista atualizada;
  • DGV:
    • O DGV terá uma coluna com operações(funções):
      • Ativar e desativar fornecedor;
      • Vizualizar contatos(s) do representante, uma vez que o contato da empresa fica à amostra no DGV;
      • Editar fornecedor;
      • Excluir fornecedor.
Produto
  • Data Grid View (DGV) com a lista de produtos (inicialmente apenas os ativos);
  • Filtro de busco por classe so produto. Ex.: Mobília - Lazer - Esporte;
  • Busca por ID do produto;
  • Campos de busca pelo nome do produto;
  • Botão adicionar produto:
    • Novo produto:
      • Deverá conter código, que poderá ser informado manualmente ou aleatoriamente.
    • Todos os campos são obrigatórios no cadastro de produto, exceto o da foto;
  • Botão listar todos os produtos no DGV (ativos e inativos);
  • Botão exibir todos os produtos caso tenha resuzido a lista devido a uma pesquisa;
  • Botão atualizar lista de produtos - em caso de mais de um usário utilizar simultaneamente o sistema e precisar da lista atualizada;
  • DGV:
    • O nome do produto é um link para carregar/abrir a imagem ilustrativa do produto, caso o mesmo tenha uma;
    • O DGV terá uma coluna com operações(funções):
      • Ativar e desativar produto;
      • Vizualizar contatos(s) do forneceodr e seu respectivo representante;
      • Editar produto;
      • Movimentar entrada/saída do produto:
        • Deve-se informar ao menos um tipo de movimentação de entrada/saída;
        • Caso de movimentação de entrada de natureza de compra, deve-se informar o fornecedor.
Ativos/Inativos
  • Colaborador:
    • Colaborador inativo fica bloqueado para logar no sistema.
  • Fornecedor:
    • Fornecedor inativo não será listado na janela de adicionar novo produto.
    • Fornecedor inativo não será listado na janela de movimentação de produto para relizar entrada de compra.
  • Produto:
    • Produto inativo fica bloqueado para realizar movimentações
